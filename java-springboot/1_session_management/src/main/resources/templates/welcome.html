STAGE 2: Your Task - Enable Redis Session Management

Option 1: Use the In-Browser Code Editor

Open Code Editor →

Option 2: Edit files in your IDE

Follow these steps to fix the problem:

Step 1: Add Redis Dependencies

  1. Open build.gradle.kts
  2. Uncomment the Redis dependencies:
    • spring-boot-starter-data-redis
    • spring-session-data-redis

Step 2: Configure Application Properties

  1. Open application.properties
  2. Change spring.session.store-type=none to spring.session.store-type=redis
  3. Uncomment the 3 Redis session configuration lines below it

Step 3: Enable Redis Session Configuration

  1. Open RedisSessionConfig.java
  2. Uncomment the @Configuration annotation
  3. Uncomment the @EnableRedisHttpSession annotation
  4. Uncomment the import statement for EnableRedisHttpSession

Step 4: Restart and Verify

  1. Restart the application: ./gradlew :1_session_management:bootRun
  2. Come back to this page and see the difference!

STAGE 3: Success - Redis Session Management Enabled!

Sessions are now stored in Redis (distributed storage)

What You Learned

  • Sessions persist across application restarts
  • Multiple application instances can share sessions (horizontal scaling)
  • Session data is stored in Redis, not in application memory
  • Enables zero-downtime deployments and load balancing
